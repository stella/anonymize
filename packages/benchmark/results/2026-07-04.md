# stella/anonymize vs. open-source PII redaction libraries

Reproducible comparison of `@stll/anonymize` against other open-source PII
detection/redaction libraries on recall, precision, and throughput over a
public, synthetic, legal-domain corpus (en/cs/de). Generated by
`bun run bench:compare` in `packages/benchmark`. See `REPRODUCING.md` for the
exact toolchain, model versions, and taxonomy-mapping decisions.

## Matching

- Primary: overlap: same label and IoU >= 0.5
- Secondary: exact: same label and identical [start, end)
- One-to-one greedy assignment (highest overlap first); a predicted span
  matches at most one gold span and vice versa.

## Overall

Two overall scores per library. `all-labels` counts every gold entity in the
corpus. `supported-labels` counts only the categories the library can emit
after taxonomy mapping, so a library is not judged on categories it never
attempts (e.g. an email-only scrubber is not charged recall on dates). Both
are reported; neither is cherry-picked.

## Throughput

Characters per second over one full corpus pass. `cold` includes first-use
lazy initialisation within the process; `warm` is a second pass at steady
state. `init` is one-time setup (model load / pipeline build) measured
separately. Python throughput is measured inside the Python process and
excludes interpreter startup. Single run on one machine: treat as
order-of-magnitude, not a precise micro-benchmark.

## Caveats and honesty notes

- **Domain skew.** The corpus is legal-domain and multilingual (en/cs/de).
  stella is built for exactly this domain and language mix, which favours it.
  Competitors tuned for English chat/email (scrubadub, redact-pii) or general
  English text (Presidio's default models) are outside their strongest
  setting on cs/de. The per-language table exists so the English-only
  comparison is visible on its own.
- **Synthetic data.** Ground truth is authored, not sampled from real
  filings, so absolute numbers may differ from production text. It is a
  fairness/consistency trade: every library sees identical inputs with exact
  gold spans, and no real personal data is published.
- **stella configuration.** stella runs its rules pipeline with NER off (the
  product default and corpus config). Enabling NER could change its numbers.
- **Mapping generosity.** Taxonomy mappings are deliberately generous to
  competitors (coarse buckets; Presidio's NRP/GPE folded into address). See
  `src/taxonomy.ts` for every decision.
- **No cherry-picking.** Every label is reported, including those where stella
  underperforms. Overlap and exact modes are both shown.
